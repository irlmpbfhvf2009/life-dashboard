# 睏寶（Kùn Bǎo）— 角色設計書

> 菜菜勇者團 第 13 位角色 · 放置炸彈流派 · 2026-07-10
> 本文件取代 2026-07-10 上線的舊版 `sleepy_melon`（該版只是「睡覺回血 + 十字 AOE」，
> 沒有放置、沒有連鎖、沒有操作決策，是典型的「換皮 AOE 角色」，整份廢棄重寫）。

---

## 1. 角色名稱：睏寶

**命名理念**：「睏」是台語/華語共通的「想睡」，「寶」既是寶貝也諧音「爆」。
睏寶 = 睏到爆，也是「一個抱著炸彈睡著的寶寶」。

名字直接編碼了他的核心機制：**他越睏（越不動），炸彈就越多越大**。
玩家看到名字 → 想到睡覺 → 玩到第一波就發現「站著不動居然變強」→ 名字與機制對上。

引信的嘶嘶聲＝他的打呼聲。這是全角色的音效母題。

---

## 2. 角色背景

菜園最角落的水塔下面，住著一顆不肯醒來的冬瓜寶寶。
他做的夢會漏出來，凝結成一顆顆會嘶嘶作響的水球炸彈。
沒有人知道他夢見了什麼，只知道他睡得越沉、夢漏得越多、炸得越響。
把他吵醒的東西，通常已經不在了。

---

## 3. 角色定位

**陷阱型 × 操作型 × 高風險高回報**（爆發來自連鎖，不來自數值）

| 軸 | 評價 |
|---|---|
| 輸出來源 | 放置物 → 連鎖爆炸（非持續 DPS） |
| 前期強度 | ★★☆☆☆（刻意偏弱，這是入場費） |
| 後期天花板 | ★★★★★（連鎖段數是指數） |
| 操作需求 | ★★★★★（全角色最高） |
| 新手友善 | ★★☆☆☆ |

**他不是 AOE 角色，他是「地形佈置」角色**：
炸彈不是攻擊，是你放在地上的「地形」。你的真正動作是**走位**——決定怪從哪裡進來、
從哪一顆開始引爆、爆風把你推去哪裡。

---

## 4. 被動技能：「夢囈引信」

睏寶**沒有一般意義的自動攻擊**。他不會朝敵人射東西。
他的 6 個武器欄組裝成**一顆炸彈的規格**，而放炸彈這個動作由**主動技能**完成（見第 5 節）。

### 4-1. 睡意（Drowsiness）0 ~ 100

| 行為 | 睡意變化 |
|---|---|
| 靜止不動 | **+30 / 秒** |
| 移動中 | **−45 / 秒** |
| 受到傷害 | **歸零**，且 0.8 秒內無法累積（驚醒） |

| 階段 | 睡意 | 效果 |
|---|---|---|
| 清醒 | 0–39 | 無 |
| 淺眠 | 40–79 | 引信 **−0.3 秒**（炸得更快） |
| 熟睡 | 80–100 | 引信 **−0.3 秒**、火力 **+1 格**、每秒回 **1.5% 最大生命** |

### 4-2. 為什麼這個被動會改變玩法

Brotato 這類遊戲的肌肉記憶是**永遠不要停下來**。
睏寶把這條規則反過來：**你必須敢站定**。

於是每一波都變成同一個問句的重複：
> 「我還能再站 1 秒嗎？」

站定 → 睡意上升 → 炸彈變多變大 → 但你正站在一堆即將爆炸的東西中間，而怪正在靠近。
這不是「數值換數值」，是**空間與時間的賭博**。

而且它天然接上放彈玩法的核心：**你就是誘餌**。
你站著睡覺，怪往你身上聚，聚到夠多的時候炸彈連鎖，爆風把你炸出包圍圈。

### 4-3. 爆風與睏寶本人

- **爆風對睏寶造成 0 傷害**
- **但會把睏寶往爆心反方向推開**（強制位移，約 260 距離）
- **被推期間獲得 0.25 秒無敵幀**

炸彈於是同時是三種東西：**傷害、地形、位移**。
高手用自己的炸彈把自己彈射穿過怪堆（炸彈跳躍），新手只是被推得東倒西歪——
但兩者都不會因此死掉。**高操作，不懲罰**。

## 5. 主動技能：「放置炸彈」（無冷卻，用儲存次數）

| 項目 | 數值 |
|---|---|
| 冷卻 | **0 秒** |
| 效果 | 在**腳下的格子**放一顆炸彈 |
| 儲存次數 | **同時炸彈上限 − 場上已有的炸彈數** |

技能鈕上的數字就是「還能放幾顆」：

```
上限 5、場上 0 顆 → 顯示 5
放 2 顆           → 顯示 3
其中一顆爆掉      → 回到 4
連放 5 顆         → 顯示 0，按不動
```

**一格只能放一顆**（炸彈對齊格心，不重疊）。這是還原度的關鍵——
你不是在「放技能」，你是在**佔格子**。場上的炸彈就是你畫在地上的地形。

**遙控器模組**接管「庫存用完時按技能」的行為：
沒有遙控器＝按不動；白階＝引爆最舊的一顆；藍階以上＝引爆全部。
於是「引爆」本身變成一個要花武器欄去換的能力。

> 註：第一版設計的主動技是「惡夢枕」（吸怪＋加速引信＋引爆全場）。
> 實作後發現它搶走了「放炸彈」這個角色最該有的動作——放彈變成自動的，玩家只是在等 CD。
> 改成「技能＝放炸彈」之後，每一顆炸彈都是玩家自己按下去的，還原度與操作感都對了。

## 6. 六把專屬武器 = 一顆炸彈的六個部位

> **這是本設計最關鍵的決定。**
> 六把武器**不是六種炸彈**，而是**同一顆炸彈的六個模組**。
> 6 個武器欄 = 你正在組裝的那顆炸彈的規格表。
>
> **六個模組沒有一個是「純傷害」**——傷害交給商店屬性、等級與連鎖。
> 六格全部拿去換玩法：數量 / 格數 / 時機 / 操作 / 引爆 / 形狀。

每個模組四階：**白 → 藍 → 紫 → 紅**（效果累積，非取代）

---

### 模組 1 ─ 彈藥箱（Ammo Crate）：同時放幾顆 → **技能的儲存次數**

| 階 | 效果 |
|---|---|
| 白 | 同時炸彈上限 **+1** |
| 藍 | **+2** |
| 紫 | **+3** |
| 紅 | **+4** |

> 連鎖流的地基。沒有炸彈數量，連鎖不會發生。

---

### 模組 2 ─ 火焰核（Flame Core）：爆風幾格（火力）

| 階 | 效果 |
|---|---|
| 白 | 火力 **+1 格** |
| 藍 | 火力 **+2 格**，爆風留下**火痕** 1.5 秒 |
| 紫 | **火痕可被爆風再次引爆**（火痕成為連鎖的介質） |
| 紅 | 火力 **+4 格**，火痕 3 秒，火痕內敵人受爆炸傷害 **+25%** |

---

### 模組 3 ─ 引信（Fuse）：幾秒後爆

| 階 | 效果 |
|---|---|
| 白 | 引信 **−0.3 秒** |
| 藍 | 引信 **−0.6 秒** |
| 紫 | **不耐煩引信**：落地 1 秒後若周圍無敵人，引信縮到 0.4 秒（沒人踩就自己炸掉） |
| 紅 | **一觸即發**：敵人碰到炸彈立即引爆 |

---

### 模組 4 ─ 踢靴（Kick Boots）：把炸彈踢出去

| 階 | 效果 |
|---|---|
| 白 | 走過去就能把自己的炸彈**踢出去滑行**，撞到牆或敵人才停 |
| 藍 | 滑行中撞到的敵人**受傷並被擊退** |
| 紫 | 踢出的炸彈**引信減半** |
| 紅 | 撞到敵人**立刻引爆** |

> 這是操作上限最高的模組：炸彈變成可以「投擲」的東西，
> 你可以把一顆快炸的彈踢進怪堆，也可以把它踢離自己腳下。

---

### 模組 5 ─ 遙控器（Detonator）：何時爆

| 階 | 效果 |
|---|---|
| 白 | 庫存用完時按技能 → **引爆最舊的一顆** |
| 藍 | 改成 **引爆全部** |
| 紫 | **保命同步**：受致命傷時引爆身邊炸彈並無敵 1 秒（每 30 秒一次） |
| 紅 | **同步爆破**：全部同時炸，且每顆分享 **15%** 傷害給其他炸彈 |

> 沒有遙控器時，庫存 0 就是真的按不動。遙控器把「等引信」換成「我決定何時爆」。

---

### 模組 6 ─ 異常核（Anomaly Core）：爆炸的形狀

| 階 | 效果 |
|---|---|
| 白 | 十字爆風（基礎形狀） |
| 藍 | 追加 **X 型**斜臂（長度 60%） |
| 紫 | 爆風**加長 40%** 且穿透敵人 |
| 紅 | 爆風末端各生一顆**子炸彈**（引信 0.6 秒、火力 −2）→ 自動擴散 |

> **子炸彈不能再生子炸彈**（無限遞迴護欄，見平衡章節）。

## 6-B. 連鎖規則（角色的心臟）

1. 爆風掃到另一顆炸彈 → **該炸彈立即引爆**
2. 連鎖第 n 段的傷害 **×(1 + 0.12 × (n−1))**
3. **段數上限預設 8**（可升級至 12；傳說「骨牌宇宙」解除上限）
4. 一次連鎖每達 **5 段**，睏寶回復 **3% 最大生命**
5. 連鎖加成只在**本次連鎖**內累積，爆完歸零

連鎖是他唯一的指數來源。所有 build 的問題都可以化約成一句：
> **「我這一輪能串幾段？」**

## 7. 推薦商店永久能力

### 新增的專屬能力（建議加進商店池）

| 能力 | 效果 | 為什麼 |
|---|---|---|
| **火力** | 爆風長度 +1 | 睏寶唯一擴大範圍的方式（他吃不到一般的「範圍」屬性） |
| **引信穩定度** | 引信時間 −8% | 快慢引信 build 的旋鈕 |
| **炸彈庫存** | 同時炸彈上限 +1 | 連鎖流的地基 |
| **連鎖上限** | 連鎖段數上限 +1 | 後期天花板 |

### 現有能力的重新解讀與聯動

| 能力 | 對睏寶的意義 | 評級 |
|---|---|---|
| 攻擊力 | 炸彈傷害（六模組沒有純傷害，這就是主要來源） | ★★★★★ |
| 固定攻擊力 | 直接加在炸彈基礎傷害上 | ★★★★★ |
| 技能冷卻 | **完全無用**——放炸彈沒有冷卻 | ☆☆☆☆☆（真陷阱） |
| 攻擊速度 | **完全無用**——他不用武器開火 | ☆☆☆☆☆（真陷阱） |
| 範圍 | 爆風臂長 | ★★★★☆ |
| 拾取範圍 | 他必須站定睡覺 → **撿不到掉落物** | ★★★★☆（隱性核心） |
| 最大生命 / 防禦 / 生命回復 | 站定當誘餌就是坦克 | ★★★★☆ |
| 暴擊率 / 暴擊傷害 | **只在爆心格生效**，需要「爆心擴大」升級才成立 | ★★☆☆☆（陷阱屬性） |
| 移動速度 | **反協同**：跑越快睡意掉越快 | ★☆☆☆☆ …除非走炸彈跳躍流 |

**刻意設計的陷阱**：移動速度與暴擊對大多數角色是萬用好屬性，對睏寶卻是負擔。
這讓「同一間商店，不同角色買不同東西」真的成立。

## 9. Build 流派（6 種）

### A. 火力流（大十字）
**核心**：火焰核 ×3 + 異常核（紫，穿透折返）+ 火力永久能力
- **優**：爆風貫穿全屏，清雜兵無敵，視覺最爽
- **缺**：單顆傷害低、庫存少 → **Boss 戰疲軟**
- **成立時機**：中期

### B. 多彈流（鋪地毯）
**核心**：彈藥箱 ×3 + 引信（白）+ 多做一顆
- **優**：場上 8~10 顆，鋪成一片，連鎖**自然發生**
- **缺**：單顆傷害極低，完全依賴連鎖加成 → **前期最弱**
- **成立時機**：中後期

### C. 快引信流（接觸引爆）
**核心**：引信（紅，一觸即發）+ 踢靴 + 貪睡鬧鐘
- **優**：把睏寶變成「近戰接觸傷害」，對衝臉怪極強，反應即時
- **缺**：炸彈一碰就炸，**留不住場面**、串不出長連鎖，總傷天花板最低
- **成立時機**：早期就成立（新手推薦）

### D. 連鎖流（畢業 build）
**核心**：彈藥箱 + 遙控器（紅，同步爆破）+ 骨牌宇宙
- **優**：**爆發天花板最高**，一次全爆可以清 Boss 一管血
- **缺**：需要鋪彈時間，被打斷就空手；對操作與節奏要求最高
- **成立時機**：後期

### E. 夢遊流（睡意坦克）
**核心**：永夜 + 厚棉被 + 夢遊 + 最大生命 / 防禦
- **優**：最穩、最省操作、自帶回血
- **缺**：輸出偏低，**怕遠程怪**（站定挨射）
- **成立時機**：全程平穩

### F. 炸彈跳躍流（機動）
**核心**：炸彈跳躍 + 移動速度 + 引信（紅）+ 爆風衝浪
- **優**：無敵幀最多、走位最帥，穿越怪堆如入無人之境
- **缺**：**數值最低**，純靠操作；與睡意系統直接對抗
- **成立時機**：只有高手能玩

## 10. 平衡性分析

### 前期（1–5 波）：**刻意偏弱**
庫存 2 顆、火力 1 格（上下左右各炸一格）、引信 2.4 秒。怪不會乖乖踩炸彈。
玩家只能站定當誘餌，把怪引到炸彈上。
**這是他的入場費**，也是「爆發型陷阱角色」的合理代價。

> 不需要修。如果他前期就強，後期的連鎖曲線會讓他變成唯一解。

### 中期（6–14 波）：**曲線陡升，這裡最危險**
模組升到藍 / 紫，連鎖開始穩定成立。

**需要限制的地方：**
- 🔴 `遙控器（紅）+ 彈藥箱（紫）` 太早到手會直接跳過整個中期
  → **紅階模組只在 wave ≥ 10 之後上架**
- 🔴 「定時同步」+「同步爆破」在中期就能做出滿段連鎖
  → 兩者其一必須是後期掉落

### 後期（15–20+）：**連鎖段數是指數來源**

段數上限是**唯一擋住溢出的東西**。
30 顆炸彈同步引爆若無上限：`×(1 + 0.12 × 30) = ×4.6`，而且互相引爆 →
傷害與實體數量同時失控。

**必要護欄：**

| 護欄 | 數值 | 理由 |
|---|---|---|
| 連鎖段數上限 | 預設 8 / 升級至 12 | 唯一的指數剎車 |
| 「骨牌宇宙」解除上限 | 佔用一個傳說欄 | 想要無限，就付出一個傳說 |
| 同時炸彈硬上限 | **12** | 伺服器實體數 + 手機渲染（本專案 Render 免費層 CPU 受限，這是硬需求） |
| 子炸彈不能再生子炸彈 | — | 無限遞迴 |
| 連鎖加成不跨次累積 | 爆完歸零 | 防止滾雪球到下一波 |
| 永夜 × 睡美人 互斥 | — | 否則睡意坦克流永遠不死 |

### 他會過強嗎？

**單人：不會。** 前期弱、操作重、遠程怪剋他。
**多人（本專案 1~4 人合作）：有風險。**
隊友幫他擋刀 → 他可以一直睡 → 睡意永遠滿。

> **多人限制建議**：睡意在「隊友替你承受傷害」時不受影響（合理），
> 但「搖籃曲」（分享睡意給隊友）必須是單向的，否則四個睏寶會互相充能。

## 11. 趣味性分析

### 為什麼會一直想玩

因為每一波都是**同一個問句的重複**，而答案每次不同：

> 「我還能再站 1 秒嗎？」

站 1 秒 = 多半顆炸彈 = 多一段連鎖 = 多 12% 傷害。
但也 = 怪多走 1 秒 = 你可能被打醒，前面全部歸零。

這是**可量化的貪婪**。跟 Roguelite 玩家的腦子直接接線。

### 爽點

**那 4 秒的表演**：

```
鋪 8 顆炸彈（站著睡覺，怪在靠近，你在賭）
    ↓
丟出惡夢枕（怪被吸成一團，引信開始瘋狂燃燒）
    ↓
第 1 顆爆 → 第 2 顆 → 第 3 顆…（螢幕開始變白）
    ↓
30 個傷害數字同時跳出來
    ↓
爆風把你自己炸飛出去（無敵幀）
    ↓
落地。怪全空。
```

這是**你自己編排的**。不是系統送你的。

### 操作技巧

1. **睡意節奏切換**：什麼時候該貪 1 秒，什麼時候該跑
2. **炸彈跳躍**：用自己的爆風 + 無敵幀穿過怪堆
3. **留一顆當引信**：故意留一顆不炸，等怪聚了讓爆風掃過去引爆
4. **枕頭落點**：那就是「我要在哪裡開一場大爆炸」的宣告
5. **引爆順序**：從外圈往內圈引爆會把怪推向中心，反之會推散

### 高手 vs 新手

| | 新手 | 高手 |
|---|---|---|
| 炸彈是什麼 | 自動傷害 | **地形** |
| 怎麼放 | 站原地一直放 | 先鋪陣型，算引爆順序 |
| 爆風 | 被推得東倒西歪 | 位移工具 + 無敵幀 |
| 睡意 | 隨便 | 精算到 0.5 秒 |
| 連鎖段數 | 2–3 段 | 8–12 段 |

**DPS 差距可以到 3~5 倍。** 這是全遊戲操作上限最高的角色。

## 12. 十個額外機制

1. **啞彈**
   5% 機率炸彈不爆，變成「啞彈」留在場上。被下一次爆風掃到時傷害 **×3**。
   → 失敗變成伏筆。

2. **入夢（debuff）**
   被爆風掃到但沒死的敵人會「打瞌睡」，2 秒內不攻擊；
   **睡著的敵人受到的爆炸傷害 +30%**。→ 連鎖自帶增傷循環。

3. **枕頭堡壘**
   主動技可長按放在腳下，變成掩體（擋遠程彈幕）。
   → 回應「夢遊流怕遠程怪」的弱點，給玩家一個選擇。

4. **數羊**
   每連鎖 1 段，畫面右上跳一隻羊。數到 100 隻 → **全場敵人強制睡著 3 秒**。
   → 一局一次的史詩時刻，且是玩家「賺來」的。

5. **夢魘事件（波次事件）**
   「他做惡夢了」——本波所有敵人變成會走路的炸彈，殺掉就爆。
   → 場地變成連鎖地獄，睏寶的主場。

6. **磁力枕**
   拾取範圍同時作用於**炸彈**——你可以把場上的炸彈吸成一堆。
   → 把「拾取範圍」這個隱性核心屬性變成顯性的 build 工具。

7. **爆風衝浪**
   連續被自己的爆風推 3 次而不落地 → 下一顆炸彈火力 **+5**。
   → 獎勵炸彈跳躍流的極限操作。

8. **搖籃曲**（多人）
   熟睡時，附近隊友也獲得睡意加成的增益。
   → 本專案是 1~4 人合作，這條把單人機制轉成團隊機制。**單向，不可互相充能。**

9. **金炸彈（隱藏）**
   單局連鎖總段數 > 500 → 解鎖金色炸彈外觀 + 金幣掉落 +10%。
   → 給連鎖流一個可炫耀的長期目標。

10. **反向引信（詛咒）**
    炸彈改為**放下時立刻爆炸**，但傷害 ×2.5、火力 −2。
    → 把睏寶整個變成近戰角色。一個選項改寫整個角色。

## 附錄：實作可行性註記（對照現有 codebase）

### 已實作（2026-07-10）

全部落地在 `game/shared/content/{characters,weapons,upgrades}.ts`、`game/shared/balance.ts` 的 `BOMB`、
新檔 `game/server/src/game/bombs.ts`，驗證腳本 `game/server/sim-kunbao.ts`（`npx tsx sim-kunbao.ts`）。

**實作時發現、設計書原本沒寫對的三件事：**

1. **放彈間隔必須明顯短於引信。**
   原本 base 間隔 2.4s / 引信 2.0s → 炸彈永遠只有一顆在地上，連鎖流與多彈流**在數學上不可能成立**。
   已改成間隔 1.1s / 引信 2.4s。

2. **被自己的爆風炸飛，不能算「移動」。**
   否則兩個核心機制直接打架：你放炸彈 → 炸彈炸飛你 → 睡意歸零 → 永遠睡不著。
   現在被炸飛時睡意照常累積（睡著的人被拋來拋去，還是在睡）。

3. **爆風剛生出來的子炸彈不能被同一次爆風引爆。**
   否則「擴散」會退化成「原地多打一次」。用 `born < g.time` 擋掉。
